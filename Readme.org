* Overview

org-re-reveal is an Org mode export backend that transforms your Org documents
into beautiful [[https://revealjs.com/][reveal.js]] presentations.

This project is a [[https://github.com/lechten/org-reveal][fork of org-reveal]] with active maintenance and
additional features.

View this README as a [[https://oer.gitlab.io/org-re-reveal/][live presentation]] to see org-re-reveal in action.

* Requirements

- GNU Emacs 24.4 or later
- Org mode 8.3 or later
- reveal.js
- htmlize or highlight.js for syntax highlighting

* Installation

** From MELPA

org-re-reveal is available on [[https://melpa.org/#/org-re-reveal][MELPA]]. After [[https://melpa.org/#/getting-started][setting up MELPA]], install
the package and add to your configuration:

#+BEGIN_SRC emacs-lisp
(require 'org-re-reveal)
#+END_SRC

Alternatively, add ~re-reveal~ to ~org-export-backends~ via customization.

** From GitLab

Clone the repository:

#+BEGIN_SRC sh
git clone https://gitlab.com/oer/org-re-reveal.git
#+END_SRC

Add the directory to your load path and require the package:

#+BEGIN_SRC emacs-lisp
(add-to-list 'load-path "/path/to/org-re-reveal")
(require 'org-re-reveal)
#+END_SRC

** Installing reveal.js

Download reveal.js from the [[https://github.com/hakimel/reveal.js/][official repository]] and extract it.

If you prefer a ready-to-use setup, consider [[https://gitlab.com/oer/emacs-reveal][emacs-reveal]], which bundles
org-re-reveal with reveal.js and popular plugins.

* Quick Start

1. Ensure org-re-reveal is loaded (via ~require~ or customization)
2. Open an Org file (try this ~Readme.org~)
3. Export with ~C-c C-e v v~ (to file) or ~C-c C-e v b~ (to file and browser)

The default reveal.js location is ~./reveal.js~ relative to your Org file.
Customize ~org-re-reveal-root~ or set ~#+REVEAL_ROOT:~ to change this.

#+BEGIN_SRC org
,#+REVEAL_ROOT: https://cdn.jsdelivr.net/npm/reveal.js
#+END_SRC

* Slide Structure

** Heading Levels (HLevel)

org-re-reveal maps Org headings to reveal.js slides using a configurable
heading level threshold:

- Headings at or above HLevel become horizontal slides
- Deeper headings become vertical slides

The default HLevel is 1, meaning only top-level headings create horizontal
slides.

*** Example

#+BEGIN_SRC org
,* Heading 1
,* Heading 2
,** Heading 2.1
,*** Heading 2.1.1
,* Heading 3
#+END_SRC

With HLevel=1, headings 2.1 and 2.1.1 become vertical slides under Heading 2:

[[./images/hlevel.png]]

#+REVEAL: split

With HLevel=2, Heading 2.1 moves to the horizontal queue:

[[./images/hlevel2.png]]

*** Setting HLevel

Globally:

#+BEGIN_SRC emacs-lisp
(setq org-re-reveal-hlevel 2)
#+END_SRC

Per file:

#+BEGIN_SRC org
,#+REVEAL_HLEVEL: 2
#+END_SRC

** Subheadings

To display a subheading on the same slide as its parent:

#+BEGIN_SRC org
,* Main Heading
,** Subheading
   :PROPERTIES:
   :UNNUMBERED: notoc
   :NOSLIDE: t
   :END:
#+END_SRC

** Splitting Slides

When content exceeds one slide, split manually:

#+BEGIN_SRC org
,#+REVEAL: split
#+END_SRC

#+REVEAL: split data-background="#555555"

You can add HTML attributes to the split:

#+BEGIN_SRC org
,#+REVEAL: split data-background="#555555"
#+END_SRC

* Appearance

** Themes and Transitions

Set globally with these options:

#+BEGIN_SRC org
,#+REVEAL_THEME: black
,#+REVEAL_TRANS: slide
,#+REVEAL_SPEED: default
#+END_SRC

Available themes are in the ~css/theme/~ directory of reveal.js. Transitions
include: none, fade, slide, convex, concave, zoom.

** Title Slide

org-re-reveal generates a title slide from your document's export settings
(title, author, email, date).

To disable it:

#+BEGIN_SRC org
,#+REVEAL_TITLE_SLIDE:
#+END_SRC

To customize, set ~org-re-reveal-title-slide~ to HTML with escape sequences
(documented in the variable's docstring) or a filename containing the HTML.

See [[https://gitlab.com/oer/org-re-reveal/blob/master/test-cases/test-title-slide.org][test-title-slide.org]] for examples.

* Fragments

Make content appear incrementally using ~ATTR_REVEAL~:

#+ATTR_REVEAL: :frag roll-in
Paragraphs can be fragmented.

#+ATTR_REVEAL: :frag roll-in
- Lists can
- be fragmented.

** Fragment Styles

Available styles:
#+ATTR_REVEAL: :frag t
* grow
* shrink
* roll-in
* fade-out
* highlight-red
* highlight-green
* highlight-blue
* appear

Using ~:frag t~ applies the default style, which can be set via
~#+REVEAL_DEFAULT_FRAG_STYLE~ or ~org-re-reveal-default-frag-style~.

** Fragment Order

Control the sequence with ~:frag_idx~:

#+ATTR_REVEAL: :frag t :frag_idx 3
This appears third.

#+ATTR_REVEAL: :frag t :frag_idx 2
This appears second.

#+ATTR_REVEAL: :frag t :frag_idx 1
This appears first.

** List Item Fragments

Apply different styles to each list item:

#+BEGIN_SRC org
,#+ATTR_REVEAL: :frag (grow shrink roll-in fade-out none) :frag_idx (4 3 2 1 -)
   * I will grow.
   * I will shrink.
   * I rolled in.
   * I will fade out.
   * I don't fragment.
#+END_SRC

#+ATTR_REVEAL: :frag (grow shrink roll-in fade-out none) :frag_idx (4 3 2 1 -)
* I will grow.
* I will shrink.
* I rolled in.
* I will fade out.
* I don't fragment.

#+REVEAL: split

Use ~:frag (appear)~ to apply the same style to all items:

#+ATTR_REVEAL: :frag (appear)
* First item
* Second item
* Third item

** Fragments with Audio

With the [[https://github.com/rajgoel/reveal.js-plugins][audio-slideshow plugin]]:

#+BEGIN_SRC org
,#+ATTR_REVEAL: :frag (appear) :audio (1.ogg 2.ogg none)
   * Appears with audio 1.ogg
   * Appears with audio 2.ogg
   * Appears without audio
#+END_SRC

* Advanced Features

** Data State
   :PROPERTIES:
   :reveal_data_state: alert
   :END:

Add a ~data-state~ attribute to trigger reveal.js behaviors or plugins:

#+BEGIN_SRC org
,** Slide Title
   :PROPERTIES:
   :reveal_data_state: alert

** Extra Slide Attributes

Add custom attributes to all slides:

#+BEGIN_SRC org
,#+REVEAL_EXTRA_ATTR: data-autoslide=20000
#+END_SRC

Or per heading:

#+BEGIN_SRC org
,** Slide Title
   :PROPERTIES:
   :reveal_extra_attr: data-autoslide=10000
   :END:
#+END_SRC

Add classes with ~:reveal_extra_attr: class="my-class"~ or use
~html_headline_class~ for the heading element itself.

** Plugins

*** Built-in Plugins

Activate reveal.js plugins:

#+BEGIN_SRC org
,#+REVEAL_PLUGINS: (notes search zoom highlight)
#+END_SRC

*** Third-Party Plugins

Configure external plugins via ~org-re-reveal-external-plugins~ (associative
list or filename). Each entry needs a ~src~ path:

#+BEGIN_SRC emacs-lisp
(setq org-re-reveal-external-plugins
      '((plugin-name . " {src: \"%srelative/path/to/plugin.js\"}")))
#+END_SRC

Or use a file with ~#+REVEAL_EXTERNAL_PLUGINS: plugins.js~.

Several plugins are pre-configured in [[https://gitlab.com/oer/emacs-reveal][emacs-reveal]].

** MathJax
  :PROPERTIES:
  :CUSTOM_ID: my-heading
  :END:

LaTeX equations render natively:

${n! \over k!(n-k)!} = {n \choose k}$

This requires an internet connection or a local MathJax installation.
Set ~REVEAL_MATHJAX_URL~ for local installations.

** Speaker Notes

Press ~s~ during a presentation to open speaker notes.

#+BEGIN_SRC org
,* Slide Title
   :PROPERTIES:
   :reveal_extra_attr: data-timing="120"
   :END:
   Content here.
,#+BEGIN_NOTES
Speaker notes go here.
- Use bullet points
- As needed
,#+END_NOTES
#+END_SRC

Configure timing globally with ~org-re-reveal-defaulttiming~.

*** Note Template

With Org 9.2+, type ~<n TAB~ to insert a notes block (requires ~org-tempo~).
Customize the key with ~org-re-reveal-note-key-char~.

** Multiplexing

Share your presentation in real-time with remote viewers.

See [[https://gitlab.com/oer/org-re-reveal/blob/master/test-cases/test-multiplex.org][test-multiplex.org]] for setup instructions. You need:

- ~multiplex~ plugin enabled
- Socket.io server URL, ID, and Secret
- Separate master (with Secret) and client presentations

** Raw HTML

Embed HTML directly in slides:

#+BEGIN_SRC org
,#+REVEAL_HTML: <video controls width="400" data-src="video.mp4"></video>
#+END_SRC

Example video (using lazy loading):

#+REVEAL_HTML: <video controls width="400" height="300" data-src="https://archive.org/download/LinusTorvaldsOnGittechTalk/LinusTorvaldsOnGittechTalk.ogv"></video>

** Preamble and Postamble

Add content before and after slides:

#+BEGIN_SRC org
,#+REVEAL_PREAMBLE: <div>Before slides</div>
,#+REVEAL_POSTAMBLE: <div>After slides</div>
#+END_SRC

For the HTML ~<head>~:

#+BEGIN_SRC org
,#+REVEAL_HEAD_PREAMBLE: <meta name="author" content="Your Name">
#+END_SRC

These can also be Emacs Lisp functions that accept Org info and return HTML.

#+BEGIN_SRC org
,#+REVEAL_EXTRA_CSS: custom-styles.css
#+END_SRC

** Extra JavaScript

#+BEGIN_SRC org
,#+REVEAL_EXTRA_SCRIPTS: ("helper.js" "<script>inline code</script>")
#+END_SRC

** Single File Export

Embed images and scripts into one portable HTML file:

#+BEGIN_SRC org
,#+OPTIONS: reveal_single_file:t
#+END_SRC

#+REVEAL: split

Limitations:

- Remote images are not embedded
- PDF export, Markdown, zoom, notes, and remote control are disabled
- Plugins are not included
- CSS must be self-contained (no ~@import~ or external URLs)
- Data URIs must be used for images in custom title slides

** Subtree Export

Export only the current subtree with ~C-c C-e v s~.

Include a title slide with ~org-re-reveal-subtree-with-title-slide~.

* Tips

** Heading Numbers

Enable or disable with the ~num~ option:

#+BEGIN_SRC org
,#+OPTIONS: num:nil
#+END_SRC

With heading numbers enabled, ~org-re-reveal-generate-custom-ids~ can generate
stable slide IDs.

** Table of Contents

Disable with:

#+BEGIN_SRC org
,#+OPTIONS: toc:nil
#+END_SRC

Place the TOC at a specific location:

#+BEGIN_SRC org
,** Table of Contents
   :PROPERTIES:
   :UNNUMBERED: notoc
   :END:
,#+REVEAL_TOC: headlines 1
#+END_SRC

See [[https://gitlab.com/oer/org-re-reveal/blob/master/test-cases/test-slide-numbers-reveal-toc.org][test-slide-numbers-reveal-toc.org]] for an example.

** Internal Links

Link to slides by headline text or CUSTOM_ID:

- [[Tips]]
- [[#my-heading][MathJax Section]]

Enable cross-presentation links with:

#+BEGIN_SRC org
,#+OPTIONS: reveal_inter_presentation_links:t
#+END_SRC

** Custom Initialization

Pass custom JavaScript to ~Reveal.initialize~:

#+BEGIN_SRC org
,#+REVEAL_INIT_SCRIPT: // Your custom JS here
#+END_SRC

** Troubleshooting head.min.js

Older reveal.js versions included ~head.min.js~. If you see errors about this
file, customize ~org-re-reveal-script-files~ to remove it.
